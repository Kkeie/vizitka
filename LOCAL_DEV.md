# Локальная разработка

## Быстрый старт

### 1. Запуск бэкенда

```bash
cd backend
npm install
npm run build
npm start
```

Бэкенд будет доступен на `http://localhost:3000`

### 2. Запуск фронтенда

В **отдельном терминале**:

```bash
cd frontend
npm install
npm run dev
```

Фронтенд будет доступен на `http://localhost:5173`

**ВАЖНО**: Используйте `npm run dev`, а не `npm run build` + `npm start` для локальной разработки!

Vite проксирует запросы `/api` на `http://localhost:3000` автоматически в dev режиме.

### 3. Первый запуск

1. Откройте `http://localhost:5173`
2. Зарегистрируйтесь или войдите
3. После входа вы сможете редактировать профиль и добавлять блоки

## Устранение проблем

### Ошибка "Не удалось загрузить данные"

1. **Проверьте, что бэкенд запущен**: откройте `http://localhost:3000/api/health` в браузере - должно вернуться `{"ok":true}`

2. **Проверьте, что фронтенд запущен в dev режиме**: используйте `npm run dev`, а не production build

3. **Проверьте консоль браузера** (F12): там должны быть логи `[API]` с информацией о запросах

4. **Проверьте авторизацию**: убедитесь, что вы залогинены (есть токен в localStorage)

5. **Проверьте CORS**: если видите ошибки CORS в консоли, убедитесь, что бэкенд разрешает запросы с `http://localhost:5173`

### Бэкенд не запускается

- Убедитесь, что порт 3000 свободен
- Проверьте, что база данных создана (должна создаться автоматически при первом запуске)

### Фронтенд не подключается к бэкенду

- Убедитесь, что используете `npm run dev` (не production build)
- Проверьте, что бэкенд запущен на порту 3000
- Откройте консоль браузера (F12) и проверьте логи `[API]`
