## Глоссарий

| Термин | Значение |
|--------|----------|
| Bearer-токен | JWT, передаётся в заголовке `Authorization: Bearer <token>` |
| Гость | Пользователь без активного токена |
| Авторизованный | Пользователь с действующим Bearer-токеном |
| Предусловие | Состояние системы до начала теста |
| ОР | Ожидаемый результат |

## Модуль: Аутентификация

---

### TC-AUTH-01
**Название:** Успешная регистрация нового пользователя  
**Тип:** Функциональный, позитивный  
**Приоритет:** Высокий  
**Метод и путь:** `POST /api/auth/register`

**Предусловие:**  
Бэкенд запущен и отвечает. Пользователь с username `newuser_01` в базе данных отсутствует.

**Шаги:**
1. Отправить запрос:
   ```
   POST /api/auth/register
   Content-Type: application/json

   { "username": "newuser_01", "password": "password123" }
   ```

**Ожидаемый результат:**
- HTTP-статус: `200`
- Тело ответа содержит поле `token` — непустая строка
- Тело ответа содержит объект `user` с полями `id`, `username`, `createdAt`, `profile`
- `user.username` равен `"newuser_01"`
- `user.profile` не `null`

   ```
   PATCH /api/profile/
   ```
