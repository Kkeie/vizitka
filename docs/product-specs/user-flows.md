# Пользовательские сценарии

## Регистрация

1. Пользователь выбирает username (latin, цифры, underscore, мин. 3 символа) + email + пароль (мин. 4 символа).
2. Backend создаёт `User` + `Profile` в транзакции, возвращает JWT.
3. Если username занят: `{ error: "username_taken", suggestions: [...] }` — до 42 альтернатив.
4. После регистрации внутри редактора запускается онбординг-мастер (`OnboardingWizard → OnboardingInEditor`).

## Редактор (`/editor`)

Авторизованная страница. Содержит:
- **Карточка профиля** — аватар, фон, имя, bio, контакты (телефон, email, telegram)
- **Сетка блоков** — добавление, редактирование, удаление, drag-to-reorder, resize
- **Публичный URL** + QR-код
- **Счётчик просмотров** за сегодня (обновляется в реальном времени)

### Управление блоками

1. Нажать "Добавить блок" → выбрать тип → заполнить форму в `BlockModal` → блок появляется в сетке
2. Перетащить блок для переупорядочивания
3. Изменить размер через `SizeMenu`
4. Нажать на блок для inline-редактирования (`InlineEditCard`) или через модальное окно

## Публичная страница (`/:username`)

Без авторизации. Отображает профиль + блоки в read-only бенто-сетке. Каждый визит инкрементирует дневной счётчик просмотров пользователя.

Также доступна по: `/u/:username`, `/public/:username`.

## Смена пароля

Требует авторизации и текущего пароля. При успехе возвращается новый JWT; все остальные сессии инвалидируются.

## Поделиться визиткой

Редактор отображает публичный URL и QR-код (`/api/qr?url=...`). Пользователь может скопировать ссылку или скачать QR.
