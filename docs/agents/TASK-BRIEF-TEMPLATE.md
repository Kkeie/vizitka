# Task Brief Template

Use this brief before medium or large tasks to improve reliability.

## Inputs

- **Goal:** one sentence expected outcome.
- **Scope:** which files/modules are in scope.
- **Out of scope:** what must not be changed.
- **Constraints:** performance/security/compatibility limits.
- **Validation required:** exact commands to run.

## Completion criteria

- Required behavior is implemented.
- Tests added or updated for new logic.
- Required validations passed (or explicit gap is reported).
- Risks and follow-ups are listed briefly.

## Optional risk flags

- touches auth,
- touches payments/billing,
- touches shared infrastructure,
- touches public API contracts.
