# Design Docs

## Основные принципы

**Простота важнее функций.**  
Один профиль, одна сетка, без кастомных доменов, без тем. Ограничение и есть продукт.

**Mobile-first сетка.**  
2 колонки на mobile/tablet, 4 на desktop. Блоки перетекают автоматически.

**Без внешнего state manager.**  
React Context + localStorage для auth. Нет Redux, нет Zustand — сложность не оправдана для масштаба проекта.

**Реальные интеграционные тесты.**  
Backend тесты работают с реальной SQLite, не с моками. Быстро (< 5с), надёжно. Моки скрывают реальные баги.

**CSS-токены вместо компонентных библиотек.**  
`var(--primary)`, `var(--accent)` и т.д. Нет Tailwind, нет MUI — полный контроль над внешним видом.

**Inline миграции, без migration runner.**  
`try/catch ALTER TABLE` в `initDatabase()`. Проще для небольшой схемы, не нужен отдельный инструмент.
