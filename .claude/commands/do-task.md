# Do Task

Берёт первую задачу из колонки «Сделать» на доске Trello и полностью выполняет её, двигая карточку по статусам.

## Колонки доски

```
Бэклог → Сделать → В работе → Тестирование → Сделано
```
«Архив» — только человек.

## Шаги

### 1. Взять задачу

- Получить список карточек из колонки **«Сделать»**.
- Взять первую (верхнюю) карточку.
- Прочитать название, описание, чеклисты и метки.
- Если колонка пустая — сообщить пользователю и остановиться.

### 2. Переместить в «В работе»

- Переместить карточку в колонку **«В работе»**.

### 3. Реализовать задачу

- Уточнить критерии приёмки из описания карточки.
- Реализовать минимально необходимое изменение.
- Добавить или обновить тесты для новой логики.

### 4. Переместить в «Тестирование»

- Переместить карточку в колонку **«Тестирование»**.

### 5. Прогнать проверки

Для каждого затронутого пакета (`backend` / `frontend`):

1. Lint — если есть скрипт (`npm run lint`); если нет — зафиксировать это явно.
2. Тесты: `npm test` (backend) или `npm run test:e2e` (frontend e2e).
3. Build: `npm run build` — если тесты не покрывают компиляцию.

Исправлять до зелёного. Максимум 3 цикла правок на блокер, затем эскалировать.

### 6. Переместить в «Сделано»

- Все проверки зелёные → переместить карточку в **«Сделано»**.
- Добавить комментарий к карточке с кратким отчётом:
  - что сделано,
  - какие проверки прогнаны и результат,
  - остаточные риски или follow-up.

### 7. Отчитаться пользователю

Короткий итог: что изменилось, что проверено, что осталось.

## Правила

- Не коммитить без явной просьбы пользователя.
- Спрашивать перед крупным рефакторингом.
- Если задача расплывчатая — уточнить у пользователя до начала реализации.
- Если прогресс встал (3+ неудачных попытки) — эскалировать с описанием блокера.
